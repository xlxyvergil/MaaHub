# 优化经验文档模块详情页的计划

## 目标
1. 移除 header 中的安装命令。
2. 移除右侧侧边栏中包含的“下载到本地”和“文件列表”。
3. 调整展示方式，使其像 wiki 一样，更方便阅读。由于经验一般主要是 markdown 文档，我们将以 markdown 渲染的方式呈现经验内容。

## 详细步骤
1. 修改 `website/src/components/ExperienceDetailApp.tsx` 文件。
    *   移除右侧表示“安装”的区域，即移除包含 clone 和 cp 命令代码及其父元素 `div.w-full.md:w-80.flex-shrink-0...`。
    *   移除 Content Tabs 区域的网格布局（原本是左侧主要内容、右侧 DownloadSection），将其改为单栏的最大宽度限制，以提供更好的阅读体验。
    *   移除 `<FileViewer files={experienceData.downloadFiles} emptyLabel={t('skill.files.empty')} />` 的导入及使用。
    *   移除 `<DownloadSection files={experienceData.downloadFiles} packageName={packageName} lang={currentLang} />` 的导入及使用。

2. 增加 Markdown 渲染：
    *   从 `experienceData.downloadFiles` 中找到入口文件 (`experienceData.readme` 或默认的 `index.md`)，提取其内容 (`content`)。
    *   使用项目已有的 `react-markdown`, `remark-gfm` 和 `rehype-highlight` 来渲染该 Markdown 内容。
    *   为了展示样式，我们可以利用项目中已经存在的 Tailwind Typography 插件（`prose`）。

3. 完善 UI 和样式：
    *   给 Markdown 的容器添加 `.prose .prose-slate .dark:prose-invert .max-w-none` 等类名。
    *   将整体的布局结构精简为一个居中的文章阅读页面。

## 验证
我们只需编辑 `website/src/components/ExperienceDetailApp.tsx` 文件，移除无关组件，加入 React Markdown 渲染组件，使用户看到一个类似于 Wiki 文档页的排版。预览效果确认符合预期。
