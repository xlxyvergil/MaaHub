# Plan: Render All Files as Tabs with Syntax Highlighting

## Context
The user requested that in detail pages for `skills`, `pipelines`, `customs`, and `experiences`, all included files (README, JSON files, source code files, etc.) should be displayed side-by-side in tabs, and code files should be syntax-highlighted. Currently, only a hardcoded `readme` and `files` tab exist, with `files` showing a hardcoded list of files instead of the actual file contents.

## Objective
1. Refactor `SkillDetailApp.tsx`, `PipelineDetailApp.tsx`, `CustomDetailApp.tsx`, and `ExperienceDetailApp.tsx` (or extract a shared component) to dynamically create tabs for every file present in the `downloadFiles` array.
2. Ensure the first tab is the README (if it exists).
3. Ensure file contents are properly displayed and code files are syntax-highlighted.
4. Support this across all four entity types (Skill, Pipeline, Custom, Experience).

## Steps

### 1. Identify File List
The files for each entity are currently read during build time and passed to the component as `downloadFiles` (or a similar prop). We'll use this array to generate tabs.

### 2. Update the Detail Components
For each detail component (`SkillDetailApp.tsx`, etc.):
- Remove the hardcoded `readme` and `files` tabs.
- Create a state `activeTab` that holds the path of the currently selected file. Default to `README.md` or `index.md` if available, otherwise the first file.
- Render a tab bar listing all file paths from `downloadFiles`.
- Render the content of the currently selected file.

### 3. Syntax Highlighting
- Astro components and React components are mixed here. Since we want client-side tabs, we'll need to render the content within the React component.
- The user requested syntax highlighting. We can use a lightweight library for React or a CSS-only approach if a library isn't already installed, or we can see what Astro provides.
- Currently, there is no code highlighting library like PrismJS, Highlight.js, Shiki, or React-Syntax-Highlighter installed in `website/package.json`.
- The user requested code files be syntax-highlighted. Adding a lightweight library to `package.json` makes sense here, given that it's a static site. Alternatively, we could install something like `prismjs` and `react-syntax-highlighter`. Since we only have pure React and Astro, and Astro's code blocks are server-side only unless we render them at build time, using `react-syntax-highlighter` inside the `SkillDetailApp` components might be the easiest way to get real-time client-side rendering with highlighting, or doing markdown parsing client-side with `react-markdown` + `remark-gfm` + `react-syntax-highlighter`.
