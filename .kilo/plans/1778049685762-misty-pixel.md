# Plan: 在 Pipeline 详情页新增 MPE 可视化模式

## 背景
用户要求基于 MPE 嵌入通信协议，在 `pipeline` 模块详情页中新增一个“可视化模式”。用户点击后，详情区域切换为 `iframe`，并向 iframe 内的 MaaPipelineEditor 注入当前 pipeline 的 JSON 数据。

## 现状调研
1. `website/src/components/PipelineDetailApp.tsx`
   - 当前仅渲染头部信息、安装命令、`FileViewer` 和 `DownloadSection`。
   - 没有 tab / mode 切换，也没有 iframe 容器。
2. `website/src/pages/pipelines/[...id]/index.astro`
   - 已通过 `withDownloadFiles(pipeline.data, 'pipelines')` 将整个 pipeline 目录下的文件读入 `downloadFiles`。
3. `website/src/lib/downloadFiles.ts`
   - 会递归读取模块目录的所有文件，并把 `{ path, content }[]` 传给详情组件。
   - 这意味着前端已经能拿到 `pipeline.json` 的原始文本。
4. `website/src/components/FileViewer.tsx`
   - 已具备多文件 tab 展示和代码高亮，但不支持“可视化模式”。
5. 仓库内未发现现有 `iframe`、`postMessage`、`mpe:` 相关实现，可视化嵌入需要新加一套最小宿主逻辑。
6. 当前 `Storage/pipelines` 中实际样例包含 `pipeline.json`，因此可先按“读取 `downloadFiles` 中的 `pipeline.json`”实现。

## 目标
1. 在 pipeline 详情页提供“文件模式 / 可视化模式”切换入口。
2. 切换到“可视化模式”后，渲染 MPE iframe。
3. iframe 加载完成后，宿主按协议发送：
   - `mpe:init`
   - 收到 `mpe:ready` 后发送 `mpe:loadPipeline`
4. 注入数据来自当前 pipeline 目录中的 `pipeline.json` 文件内容。
5. 当不存在 `pipeline.json` 或 JSON 解析失败时，提供清晰的降级提示，不影响原有文件预览模式。

## 建议实现方案

### 1. 在 `PipelineDetailApp.tsx` 增加模式状态
新增本地状态，例如：
- `viewMode: 'files' | 'visual'`
- `visualError: string | null`
- `pipelineContent` / `pipelineData`

处理逻辑：
- 默认进入 `files` 模式，保持现有行为不变。
- 用户点击“可视化模式”后再渲染 iframe，避免页面初次加载即访问第三方站点。

### 2. 从 `downloadFiles` 中提取 `pipeline.json`
在 `PipelineDetailApp` 中从 `pipelineData.downloadFiles` 查找目标文件：
- 优先精确匹配 `pipeline.json`
- 如果未来支持子目录，可优先取文件名为 `pipeline.json` 的项

解析策略：
- 找到后执行 `JSON.parse(file.content)`
- 解析成功则保存对象用于 `mpe:loadPipeline.payload.data`
- 解析失败则禁用/阻止进入可视化模式，并显示错误文案

### 3. 新增最小可复用的 MPE 嵌入组件
建议新增一个专用 React 组件，例如 `PipelineVisualizer.tsx`，职责单一：
- 渲染 iframe
- 管理 `load` / `message` 生命周期
- 执行握手与数据注入
- 展示 loading / error 状态

建议 props：
- `fileName: string`
- `pipeline: unknown`
- `lang?: 'zh' | 'en'`

这样 `PipelineDetailApp` 只负责模式切换和数据准备，嵌入协议逻辑集中在一个组件中。

### 4. 按协议实现宿主侧通信
嵌入 URL：
- `https://mpe.codax.site/stable/?embed=true&origin=maahub`

宿主消息实现建议：
1. iframe `load` 后发送 `mpe:init`
2. `mpe:init` 内容使用保守配置，优先满足“查看可视化”需求：
   - `capabilities.readOnly: true`
   - `allowCopy: false` 或 `true` 均可，推荐 `false`
   - `allowUndoRedo: false`
   - `allowAutoLayout: true` 或 `false`，推荐 `true`
   - `allowAI: false`
   - `allowSearch: true`
   - `allowCustomTemplate: false`
3. `ui` 建议尽量精简，减少与 MaaHub 页面双重导航冲突：
   - `hideHeader: true`
   - `hideToolbar: false`
   - `hiddenPanels` 至少隐藏与本地服务、日志、探索等无关区域
4. 收到 `mpe:ready` 后，发送 `mpe:loadPipeline`
5. 收到 `mpe:loadResult` 后更新宿主状态
6. 收到 `mpe:error` 时显示错误提示

消息安全策略：
- 仅处理 `event.data?.protocol === 'mpe-embed'`
- 可使用非严格标识符 `origin=maahub`，避免当前站点 origin 不固定时握手失败
- 发送时仍可使用 `postMessage(..., '*')`，因为严格 origin 校验由 iframe 端决定

### 5. UI 展示方式
建议在当前主内容区增加一组模式按钮，而不是修改页头结构过多：
- “文件模式”
- “可视化模式”

渲染规则建议：
- `files` 模式：保留现有 `FileViewer`
- `visual` 模式：以卡片容器承载 iframe
- iframe 容器应提供固定最小高度，如 `min-h-[70vh]` 或类似高度，避免编辑器被压扁

### 6. 降级与异常处理
需要覆盖以下情况：
1. 未找到 `pipeline.json`
   - “可视化模式”按钮置灰，或点击后显示提示
2. `pipeline.json` 不是合法 JSON
   - 显示解析失败信息
3. 第三方 iframe 加载失败 / 握手失败 / 超时
   - 显示“可视化加载失败，请改用文件模式查看”之类的提示
4. 用户切回文件模式
   - 停止展示 iframe；是否卸载 iframe 由 React 条件渲染自然完成

### 7. 文案与国际化
需要在 `website/src/i18n/utils.ts` 中补充至少这些文案：
- `pipeline.view.files`
- `pipeline.view.visual`
- `pipeline.visual.loading`
- `pipeline.visual.unavailable`
- `pipeline.visual.invalidJson`
- `pipeline.visual.handshakeFailed`
- `pipeline.visual.loadFailed`

如果不想单独扩写大量 pipeline 专属 key，也可采用更通用的 `common.visual` / `common.source` 等 key，但要保持命名清晰。

## 预期修改文件
1. `website/src/components/PipelineDetailApp.tsx`
   - 增加模式切换
   - 提取并解析 `pipeline.json`
   - 接入可视化组件
2. `website/src/components/PipelineVisualizer.tsx`
   - 新增 MPE iframe 与 postMessage 逻辑
3. `website/src/i18n/utils.ts`
   - 增加可视化模式相关中英文文案
4. 可能无需修改 `website/src/pages/pipelines/[...id]/index.astro`
   - 除非需要额外透传 props；按当前结构大概率不用动

## 实施顺序
1. 在 `PipelineDetailApp.tsx` 中确认并抽取 `pipeline.json` 数据源。
2. 新建 `PipelineVisualizer.tsx`，实现 iframe、握手、消息监听、错误状态。
3. 回到 `PipelineDetailApp.tsx`，接入模式切换 UI 和条件渲染。
4. 在 `i18n` 中补充相关文案。
5. 做静态检查与页面验证，重点验证：
   - 默认仍为文件模式
   - 有 `pipeline.json` 时可以切换并完成渲染
   - 缺失/非法 JSON 时有降级提示

## 验证要点
1. 打开任一 pipeline 详情页，默认显示文件预览。
2. 点击“可视化模式”后，iframe 正常出现。
3. MPE 完成 `mpe:init -> mpe:ready -> mpe:loadPipeline -> mpe:loadResult` 流程。
4. 页面刷新后，文件模式仍可正常工作。
5. 在 `pipeline.json` 不存在或内容非法时，不会导致整个详情页崩溃。

## 风险与注意事项
1. 第三方站点 `https://mpe.codax.site/stable/` 的可用性会直接影响可视化模式。
2. 若未来 pipeline 文件不是 `pipeline.json` 命名，而是多文件组合方案，需要扩展查找规则。
3. `client:only="react"` 页面里所有握手逻辑都在客户端执行，不能依赖 SSR。
4. 如果 MPE 对 `mpe:init` 的时机较敏感，需在 iframe `load` 后短暂延时发送或重试一次；实现时可先按一次发送，必要时加超时保护。

## 已确认范围
1. 保留站点 Header，但在“可视化模式”下使用简化版详情头部与精简的 pipeline 基础信息。
2. 页面的大部分垂直空间优先留给 iframe，而不是文件列表或大块说明区。
3. 实现上优先采用“主体区切换”为 iframe，而不是只在小卡片内嵌一个编辑器窗口。

## UI 落地补充
1. `files` 模式沿用当前两栏布局。
2. `visual` 模式切成更偏单栏的大画布布局：
   - 保留返回入口、标题、作者/版本等少量必要信息
   - 弱化安装命令和侧边下载区，避免挤占可视区域
   - iframe 容器尽量占满内容宽度，并提供较大的最小高度
3. 如果现有 `PipelineDetailApp` 结构不利于切换，允许在组件内部对 `visual` 模式做条件化布局，而不是强行复用当前三栏结构。
